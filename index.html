Task Management App

I developed a task management web application using ASP.NET Core MVC (.NET 8) and Bootstrap 5, focused on implementing clean CRUD functionality in a structured MVC architecture. The application allows users to create, view, edit, and delete tasks through a responsive interface, with each task containing a title, description, status, due date, and created-by field. I implemented server-side validation using Data Annotations, proper error handling with 404 responses for missing tasks, and status visualization with Bootstrap badges. To match the project requirements, I used in-memory storage instead of a database and separated responsibilities through Models, Services, Controllers, and Views, while also applying Dependency Injection for the service layer. This project helped me practice ASP.NET Core MVC, routing, model binding, validation, service-layer design, dependency injection, Razor views, and building a complete responsive web application from scratch. Word Game

I developed a Wordle-style word guessing game using HTML, CSS, and vanilla JavaScript, fully powered by asynchronous API calls. The game fetches a random five-letter word from an external API and validates each user guess in real time. I implemented a full guessing system with six rounds, a dynamic letter grid, and color-coded feedback that marks letters as correct, close, or wrong by comparing guess frequency using a custom mapping function. The app includes smooth interactions such as backspace handling, invalid word animations, loading indicators controlled by an async state toggle, and win/lose logic. This project helped me practice asynchronous JavaScript (async/await), API integration, DOM manipulation, keyboard event handling, and building game logic from scratch without libraries. Mole Game

I created an interactive Whac-A-Mole–style browser game using HTML, CSS, and vanilla JavaScript. The game features multiple mole states (hungry, fed, sad, leaving) that update dynamically based on timing functions I implemented. Each mole is controlled through a state-machine-like system, and players earn points by clicking the hungry moles as they pop up. The game uses animation frames for smooth updates, event listeners for user interaction, and a visual score meter that fills as you progress. When the player reaches the winning score, the UI switches to a victory screen. This project helped me practice DOM manipulation, timing logic, and building interactive front-end features without frameworks. Explore the code!

Calculator

I built a fully functional calculator app using HTML, CSS, and vanilla JavaScript. The core logic is based on managing a number buffer, tracking the running total, and handling operators through a simple state-machine structure. Each button click is processed through event delegation, allowing the calculator to efficiently interpret numbers, mathematical symbols, and special actions like clear and backspace. The JavaScript handles parsing, arithmetic operations, and live screen updates, creating a smooth, responsive calculator experience without using any external libraries. This project helped me strengthen my understanding of DOM manipulation, event handling, and clean, modular logic for front-end applications. Explore the code!
